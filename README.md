## Publish with GitHub Pages — browser-only method

1. Sign in to GitHub with the `magicmelonG` account.
2. Click the `+` menu in the top-right corner and choose `New repository`.
3. Name the repository exactly:

   `magicmelonG.github.io`

4. Set the repository to `Public`, then create it.
5. In the new repository, click `Add file` → `Upload files`.
6. Upload all of the following items while keeping the folder structure unchanged:

   - `index.html`
   - `styles.css`
   - `.nojekyll`
   - the `assets` folder containing `profile.jpg`

7. Commit the uploaded files to the `main` branch.
8. Open `Settings` → `Pages`.
9. Under `Build and deployment`, choose:

   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/(root)`

10. Click `Save` and wait a few minutes.
11. Open:

   `https://magicmelong.github.io/`

12. After the page is visible, paste that URL into the `Homepage URL` field on OpenReview.

## Update the page later

Edit `index.html` to change text. Replace `assets/profile.jpg` to change the portrait. After committing the changes, GitHub Pages will redeploy the site automatically.

## Files

- `index.html`: page content
- `styles.css`: visual design
- `assets/profile.jpg`: portrait cropped from the supplied resume image
- `.nojekyll`: tells GitHub Pages to serve the static files directly
